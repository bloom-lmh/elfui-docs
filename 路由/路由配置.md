# 路由配置

`RouteRecord` 支持路径、命名路由、元数据、重定向、别名、子路由、路由级守卫、props 和命名视图。

```ts
const routes = [
  {
    path: "/users/:id",
    name: "user",
    component: () => import("./pages/User"),
    props: true,
    meta: { requiresAuth: true }
  },
  { path: "/member/:id", redirect: { name: "user" } },
  { path: "/me", alias: "/profile", component: ProfilePage }
];
```

## mode

| mode      | 说明                                    |
| --------- | --------------------------------------- |
| `hash`    | 默认值，适合静态部署                    |
| `history` | 需要服务端把未知路径回退到 `index.html` |
| `memory`  | 测试或非浏览器环境                      |

## 嵌套路由

```ts
{
  path: "/settings",
  component: SettingsLayout,
  children: [
    { path: "", component: SettingsHome },
    { path: "profile", component: ProfilePage }
  ]
}
```

父页面通过 `<elf-router-view depth="1">` 放置子路由出口。

## Route props

`props: true` 把 params 传给页面；也可以传静态对象或根据 route 计算的函数：

```ts
{
  path: "/reports/:id",
  component: ReportPage,
  props: (route) => ({ id: route.params.id, compact: route.query.compact === "1" })
}
```

## 命名视图

一个 record 可以使用 `components` 渲染多个出口：

```ts
{
  path: "/workspace",
  components: {
    default: WorkspacePage,
    aside: WorkspaceAside
  }
}
```

```html
<elf-router-view></elf-router-view> <elf-router-view name="aside"></elf-router-view>
```

## 动态路由与滚动

`addRoute()` 返回移除函数，也可使用 `removeRoute(name)`、`hasRoute(name)` 和 `getRoutes()` 管理路由表。`scrollBehavior` 可返回 `{ top, left }` 或 `{ el }`，用于导航后的滚动恢复。

命名路由支持 module augmentation 类型收窄；当前需要手动声明 `RouteNamedMap`，不会从 routes 自动推导。
