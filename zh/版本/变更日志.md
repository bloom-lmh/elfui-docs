# 变更日志

## v0.1.0-beta.9 — 2026-07-22

- 修复宏 codegen 未把 `RenderContext.directives` 传给 `resolveDirective()`，局部指令现在可以在根模板和 `v-for` 子上下文中正常执行。
- `defineDirective` 正式统一为单参数形式：`const loading = defineDirective(definition)`；变量名会转换为 kebab-case 模板名，旧的双参数形式不再支持。
- 同名指令继续遵循组件局部优先于当前 App 的规则。
- 组件卸载现在会释放仍存活的 `v-if`/`v-for` detached scope，保证指令清理钩子执行。

## v0.1.0-beta.8 — 2026-07-22

### 生命周期资源所有权与 API 收口

- `onMounted()` 可以同步或异步返回清理函数。卸载时按后注册先清理的顺序，在 `onBeforeUnmount` 之后、组件 DOM 和作用域释放之前执行；迟到的异步清理会立即执行。
- 公开 API 统一为 `onMounted`、`onUnmounted`、`useComputed`、`useEffect`、明确数据源的 `watch`、`theme`、`defineDirective` 和 `app.directive`。
- 删除 beta 别名 `onMount`、`onUnmount`、`computed`、`watchEffect`、`watchPostEffect`、`watchSyncEffect`、`useTheme`，以及公开的进程级全局 `directive()` 注册表。局部指令使用 `defineDirective()`，应用级指令使用 `app.directive()`。
- 增加生命周期清理回归测试和公开 API 边界检查。

### 性能

- 长度相同且 key 顺序不变的列表现在直接更新 item/index state，不再分配完整 keyed diff 结构，也不会移动 DOM。
- 宏生成组件会懒缓存并克隆较大的纯静态原生 HTML/SVG 子树；指令、ref、组件、自定义元素、slot 和动态边界仍走常规路径。
- 没有注册生命周期 hook 的组件共享冻结空 hook 表，并在首次注册时写时复制，避免每个实例固定创建 11 个空数组。
- 动态 style binding 复用两个声明缓冲区，重复更新不再为每轮更新和每个字符串片段新建 `Map`。
- Core 全量入口已完成模块归因；完整聚合预算调整为 gzip 16.5 KB / Brotli 14.9 KB，真实 tree-shaken 应用预算仍为 gzip 9.8 KB / Brotli 8.9 KB。
- `verify:release` 现在强制执行体积预算和真实 Chromium 相对性能门禁，并继续执行外部工具、宿主框架、多 Runtime 与真实发布包消费验证。

### 迁移

直接更新 import。对于在 `onMounted` 中创建的外部资源，推荐直接返回清理函数：

```ts
onMounted(() => {
  const chart = createChart(canvas.value!);
  return () => chart.destroy();
});
```

为保持原默认调度时序，将 `watchEffect(fn)` 迁移为 `useEffect(fn, { flush: "pre" })`；原来的两个调度快捷入口分别改用 `post` 和 `sync`。

## v0.1.0-beta.7 — 2026-07-22

### 宏语法去冗余

- 新增 `defineHtml(\`...\`)`，模板中的 `${...}` 继续按响应式表达式编译。
- 新增 `defineStyle(\`...\`)`，并支持 `defineStyle(styleA, styleB)` 组合多个导入样式。
- 删除 `html`、`css`、`MacroHtmlTemplate` 的公开导出、源码实现和旧语法编译分支；这是 beta.7 的破坏性变化。
- 删除已退出公共 API 的 `useName`、`useProps`、`useEmit`、`useStyle` 编译器兼容诊断路径，并清理残留 `useState` 注释示例。
- 动态生成的任意 HTML 字符串仍不支持，框架没有因此增加运行时模板编译器。

### 迁移

Core 与 Vite 插件必须一起升级：

```bash
pnpm up @elfui/core@0.1.0-beta.7 @elfui/vite-plugin@0.1.0-beta.7
```

删除 `html`、`css` import，并把 tagged-template 直接移入宏调用：

```ts
import { defineHtml, defineStyle } from "@elfui/core";

defineStyle(`:host { display: block; }`);
export const Button = defineHtml(`<button><slot></slot></button>`);
```

外部样式仍可使用 `defineStyle(baseStyle, themeStyle)`。升级后继续使用旧 import 会产生 TypeScript 或打包错误。

本页记录 ElfUI 框架固定版本组中面向用户的变化，包括 `@elfui/shared`、`@elfui/reactivity`、
`@elfui/runtime`、`@elfui/compiler-template`、`@elfui/compiler`、`@elfui/core` 和
`@elfui/vite-plugin`。

ElfUI 目前仍处于 beta 阶段。请让 `@elfui/core` 和 `@elfui/vite-plugin` 始终使用完全相同的版本。
当版本涉及编译器与运行时协议变化时，不支持混用不同版本。

## v0.1.0-beta.6 — 2026-07-21

### 统一应用入口

- 普通应用和组件库现在只需要把 `@elfui/core` 声明为直接运行时依赖。
- Core 统一提供稳定的 runtime、响应式、生命周期、模型、指令、插件和内置渲染 API。
- 编译器生成的 helper 改为使用 `@elfui/core/internal`。该子入口属于编译器内部协议，业务代码不得手写导入。
- `@elfui/runtime`、`@elfui/reactivity` 和 `@elfui/shared` 仍会独立发布，供底层或脱离组件运行时的场景使用；普通应用不需要直接声明它们。

```bash
pnpm add @elfui/core@0.1.0-beta.6
pnpm add -D @elfui/vite-plugin@0.1.0-beta.6
```

### 公共 API 与类型修正

- `@elfui/core` 现在公开 `directive()`，以及 `onMounted()`、`onUnmounted()` 等生命周期 API。
- 可写 Ref、计算值和模型的 `set()` 返回值统一为 `void`，可以直接用于声明返回 `void` 的回调。
- `defineEmits<Interface>()` 支持有限事件接口和只读事件名称数组。
- `defineExpose<Interface>()` 支持普通有限键接口，不再要求字符串索引签名。
- 改进指令泛型、基础类型 prop 构造器、模型值和严格可选属性的类型推断。
- 运行时 `useModel()` 组合式函数不再被编译器误判为已移除的宏别名。

### 编译器与集成稳定性

- 编译产物的 helper 与公开 Core 入口之间建立了统一、可版本化的内部边界。
- 加强模板引用、事件契约、列表/控制流值和源码诊断的回归测试。
- 发布门禁覆盖外部 Canvas/SVG/WebGL 工具、Floating UI 类浮层、Observer、Worker/WASM 资源、重复挂载卸载清理以及多 runtime 副本。
- 在真实浏览器中验证原生 Custom Elements，以及 React、Vue、Svelte 和 Angular 宿主集成契约。

### 升级说明

Core 和 Vite 插件必须一起升级，然后重新安装依赖并清理 Vite 依赖缓存：

```bash
pnpm up @elfui/core@0.1.0-beta.6 @elfui/vite-plugin@0.1.0-beta.6
pnpm install
```

如果还残留旧的预构建结果，请删除 `node_modules/.vite` 或重启 Vite。以下报错通常意味着项目正在混用 beta.6 源码和更早的包：

- `does not provide an export named 'directive'`
- `Found ElfUI component pragma, but no import from "elfui"`

当 Core 和插件都升级到 beta.6 后，宏组件应从 `@elfui/core` 导入；不要为了绕过旧插件警告而改回旧的 `elfui` 导入。
