# 路由配置

`RouteRecord` 支持路径、名称、元数据、重定向、别名、子路由、守卫、props 和命名视图。

```ts
const routes = [
  {
    path: "/users/:id",
    name: "user",
    component: () => import("../pages/User"),
    props: true,
    meta: { requiresAuth: true }
  },
  { path: "/member/:id", redirect: { name: "user" } },
  { path: "/me", alias: "/profile", component: ProfilePage }
];
```

子路由会合并父级 `meta`，同名键以子级为准。静态路径的优先级高于动态路径；参数支持 `?`、`*`、`+` 和 `:id(\\d+)` 这样的自定义正则。

## History 与 base

新项目推荐使用与 Vue Router 一致的 history factory：

```ts
import {
  createMemoryHistory,
  createRouter,
  createWebHashHistory,
  createWebHistory
} from "@elfui/router";

createRouter({ history: createWebHistory("/app"), routes });
createRouter({ history: createWebHashHistory(), routes });
createRouter({ history: createMemoryHistory(), routes });
```

原有的 `mode: "history" | "hash" | "memory"` 仍然可用。`history` 模式需要服务端把未知路径回退到 `index.html`；`hash` 适合静态部署；`memory` 适合测试和非浏览器环境。`router.resolve()` 返回的 `href` 会包含配置的 base。

`sensitive: true` 开启大小写敏感匹配；`strict: true` 让尾部斜杠参与匹配。

## 嵌套、props 与命名视图

父页面使用 `<elf-router-view depth="1">` 放置子路由出口。`props: true` 传递 params；也可以传静态对象或根据 route 计算 props 的函数。

```ts
{
  path: "/workspace",
  components: { default: WorkspacePage, aside: WorkspaceAside },
  props: { default: { compact: false }, aside: { heading: "工具" } }
}
```

```html
<elf-router-view></elf-router-view>
<elf-router-view name="aside"></elf-router-view>
```

## 动态路由与滚动

`addRoute()` 返回移除函数；同名路由会替换旧记录。还可以使用 `removeRoute(name)`、`clearRoutes()`、`hasRoute(name)` 和 `getRoutes()` 管理路由表。`scrollBehavior` 可返回 `{ top, left }`、`{ el }` 或 Promise；浏览器及 memory history 的前进/后退会传入保存的滚动位置。
