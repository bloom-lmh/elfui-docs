# 生态总览

ElfUI 让框架核心保持聚焦，把体量更大、迭代节奏不同的能力放到独立项目中。新的 Macro 应用通常从脚手架开始，再按需加入其它部分。

```bash
pnpm create elfui@beta my-app --install
```

| 项目 | 职责 | 什么时候使用 |
| --- | --- | --- |
| [ElfUI](https://github.com/bloom-lmh/elfui) | 框架核心 | Macro 组件、响应式、runtime、编译器和 Vite 集成。 |
| [create-elfui](https://github.com/bloom-lmh/create-elfui) | 项目脚手架 | 创建 Vite 应用，并按需加入 Router 和质量工具。 |
| [ElfUI Router](https://github.com/bloom-lmh/elfui-router) | 路由 | 需要客户端导航，但不希望 Router 进入核心包。 |
| [ElfUI Kit](https://github.com/bloom-lmh/elfui-kit) | UI 组件库 | 使用官方组件构建产品界面。 |
| [ElfUI Extensions](https://github.com/bloom-lmh/elfui-extensions) | 可选扩展 | 使用 Chain 与未来的平台集成。 |
| [ElfUI Language Tools](https://github.com/bloom-lmh/elfui-language-tools) | 编辑器工具 | 使用 VS Code 插件和语言服务。 |
| [elfui-docs](https://github.com/bloom-lmh/elfui-docs) | 文档站 | 查阅完整指南与 API 参考。 |

新项目的默认路线是 Macro 组件加 `@elfui/vite-plugin`。Chain 是给旧页面、无构建场景和链式 builder 模型准备的扩展，不是第二条默认主线。
